The `azure-openai` provider supports the Azure OpenAI `/chat/completions` endpoint.

<Note>
  If you're using the new Microsoft Foundry portal, use the [`microsoft-foundry`](https://docs.boundaryml.com/docs/snippets/clients/providers/microsoft-foundry) provider instead.
</Note>

## Quick Start

```baml
client<llm> MyClient {
  provider "azure-openai"
  options {
    resource_name "my-resource-name"
    deployment_id "my-deployment-id"
    api_version "2024-02-01"
    api_key env.AZURE_OPENAI_API_KEY
  }
}
```

<Warning>
  `api_version` is required. Azure will return "not found" if the version is not specified.
</Warning>

## Authentication

Set your Azure OpenAI API key as an environment variable:

```bash
export AZURE_OPENAI_API_KEY="your-api-key-here"
```

Or specify it explicitly in your BAML configuration:

```baml
client<llm> MyClient {
  provider "azure-openai"
  options {
    resource_name "my-resource-name"
    deployment_id "my-deployment-id"
    api_version "2024-02-01"
    api_key env.MY_AZURE_KEY
  }
}
```

## Configuration Options

### BAML-Specific Options

These options modify the API request sent to Azure OpenAI.

<ParamField path="api_key" type="string" default="env.AZURE_OPENAI_API_KEY">
  Injected via the `API-KEY` header.
</ParamField>

<ParamField path="resource_name" type="string" required>
  Your Azure OpenAI resource name. Used to construct the base URL.
  
  The base URL is constructed as:
  ```
  https://${resource_name}.openai.azure.com/openai/deployments/${deployment_id}
  ```
</ParamField>

<ParamField path="deployment_id" type="string" required>
  Your Azure OpenAI deployment ID. This is the name you gave your model deployment in the Azure portal.
</ParamField>

<ParamField path="api_version" type="string" required>
  The Azure OpenAI API version. Passed as a query parameter `api-version`.
  
  Common versions:
  - `2024-02-01`
  - `2023-12-01-preview`
  
  See [Azure API versioning](https://learn.microsoft.com/en-us/azure/ai-services/openai/api-version-deprecation) for available versions.
</ParamField>

<ParamField path="base_url" type="string">
  Alternative to `resource_name` and `deployment_id`. Specify the full base URL directly:
  
  ```baml
  client<llm> MyClient {
    provider "azure-openai"
    options {
      base_url "https://my-resource-name.openai.azure.com/openai/deployments/my-deployment-id"
      api_version "2024-02-01"
      api_key env.AZURE_OPENAI_API_KEY
    }
  }
  ```
</ParamField>

<ParamField path="headers" type="object">
  Additional headers to send with requests.
  
  ```baml
  client<llm> MyClient {
    provider "azure-openai"
    options {
      resource_name "my-resource-name"
      deployment_id "my-deployment-id"
      api_version "2024-02-01"
      api_key env.AZURE_OPENAI_API_KEY
      headers {
        "X-My-Header" "my-value"
      }
    }
  }
  ```
</ParamField>

## Model Parameters

These parameters are passed directly to the Azure OpenAI API.

<Warning>
  For reasoning models like `o1` or `o1-mini`, use `max_completion_tokens` instead of `max_tokens`. Set `max_tokens` to `null`.
  
  See the [OpenAI API documentation](https://platform.openai.com/docs/api-reference/chat/create#chat-create-max_completion_tokens) for details.
  
  ```baml
  client<llm> AzureO1 {
    provider "azure-openai"
    options {
      resource_name "my-resource"
      deployment_id "o1-mini"
      api_version "2024-02-01"
      max_tokens null
      max_completion_tokens 4096
    }
  }
  ```
</Warning>

Common parameters:

- `temperature` - Controls randomness (0-2)
- `max_tokens` - Maximum tokens to generate
- `top_p` - Nucleus sampling parameter
- `frequency_penalty` - Reduces repetition (-2.0 to 2.0)
- `presence_penalty` - Encourages new topics (-2.0 to 2.0)

For all options, see the [Azure OpenAI API documentation](https://learn.microsoft.com/en-us/azure/ai-services/openai/reference#chat-completions).

## Finding Your Configuration

In the Azure Portal:

1. Go to your Azure OpenAI resource
2. Navigate to "Keys and Endpoint"
3. Find:
   - **Resource name**: In the endpoint URL (e.g., `https://YOUR-RESOURCE.openai.azure.com/`)
   - **API key**: Under "Key 1" or "Key 2"
4. Navigate to "Model deployments" to find your **deployment_id**

## Features

- **Streaming**: Supported for real-time response generation
- **Multimodal**: Support depends on your deployed model
- **Azure Integration**: Works with Azure AD authentication and private endpoints
- **Regional Deployment**: Deploy in your preferred Azure region
